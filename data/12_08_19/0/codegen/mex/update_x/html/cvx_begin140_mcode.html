 3%CVX_BEGIN    Starts a new CVX specification.
 4%   CVX_BEGIN marks the beginning of a new CVX model. Following this command
 5%   may be variable declarations, objective functions, and constraints, and
 6%   a CVX_END to mark the completion of the model.
 7%
 8%   If another model has already been created and is still in progress, then
 9%   CVX_BEGIN will issue a warning and clear the previous model.
10%
11%   CVX_BEGIN SDP marks the beginning of a semidefinite programming (SDP) model.
12%   This command alters the interpretation of inequality constraints when used
13%   with matrices, so that SDPs are easier to construct. Specifically, 
14%   constraints of the form
15%       X >= Y    X > Y    Y < X    Y <= X
16%   where X and Y are matrices (i.e., not vectors or scalars), CVX will
17%   interpret them all as LMIs, and convert them to
18%        X - Y == semidefinite(size(X,1));
19%   X and Y _must_ be square and identically sized---with one exception: 
20%   X or Y may also be the scalar number zero, so that expressions such as
21%   X >= 0 have the expected meaning.
22%
23%   CVX_BEGIN GP marks the beginning of a geometric programming (GP) model. This
24%   command alters the definition of the VARIABLE keyword to create geometric
25%   variables by default GP and SDP cannot be supplied simultaneously.
26%
27%   CVX_BEGIN SET can be used to mark the beginning of a set definition---a cvx
28%   feasibility problem intended to describe a set for use in other models. See
29%   the files in the cvx subdirectory sets/ for examples. The SET keyword can be
30%   combined with SDP or GP to specify sets which use SDP or GP conventions;
31%   for example, CVX_BEGIN SET SDP
32%
33%   CVX_BEGIN SEPARABLE gives permission for CVX to solve a multiobjective
34%   problem simply by taking the sum of the objectives and solving the resulting
35%   single-objective problem. As the name implies, this produces equivalent 
36%   results only when the subproblems are separable. Behavior is undefined when
37%   one or more of the subproblems is infeasible or unbounded. The keyword is
38%   ignored for sets and incomplete specifications.
39
40if ~iscellstr( varargin ), error( 'Arguments must be strings.' ); end
41assignin( 'caller', 'cvx_problem', cvxprob( varargin{:} ) );
